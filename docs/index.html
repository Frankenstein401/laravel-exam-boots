Instalasi & Pengenalan

Fungsi

Memasang package generator ke dalam project Laravel lokal kamu sehingga perintah-perintah Artisan baru siap digunakan.

Kapan Digunakan

Langkah pertama setelah kamu menginisialisasi project Laravel baru saat ujian, sebelum menulis kode fitur apa pun.

Laravel Exam Boots menggunakan konsep ejection (shadcn/ui philosophy). Kode-kode generator langsung diletakkan di dalam folder `app/` agar kamu memiliki kontrol 100% saat kustomisasi soal ujian.

Langkah Instalasi

  1. Jalankan composer require di root project Laravel kamu:
    composer require franken/laravel-exam-boots
  2. Pastikan package ter-load dengan benar:
    php artisan list | grep exam

Pre-flight Environment Doctor

Fungsi

Melakukan diagnosa komprehensif pada konfigurasi file .env, versi PHP, koneksi database, antrean migrasi, dan tautan folder storage publik.

Kapan Digunakan

Segera setelah instalasi package dan konfigurasi database selesai, serta kapan saja jika terjadi error koneksi database atau upload file tidak terbaca.

Sebelum mulai ujian, pastikan lingkungan development kamu sudah siap dengan perintah `forge:doctor`. Perintah ini meminimalkan potensi kegagalan di tengah jalan akibat masalah konfigurasi DB atau PHP.

php artisan forge:doctor

Alur Langkah Pengerjaan Ujian

Fungsi

Panduan kronologis pengerjaan modul ujian agar project tersusun rapi, aman dari error, dan selesai tepat waktu.

Kapan Digunakan

Dibaca dan dipraktikkan secara runut dari awal hingga akhir sesi ujian sertifikasi kamu.

Ikuti langkah taktis berikut untuk menyelesaikan pengerjaan boilerplate ujian secara instan, aman, dan 100% siap di-test:

Langkah 1

Pre-flight Check & Inisiasi Database

Jalankan php artisan forge:doctor untuk memastikan PHP, konfigurasi `.env`, dan koneksi database kamu terhubung dengan benar.

Langkah 2

Setup Autentikasi & Akun Awal

Setup authentication project kamu menggunakan php artisan forge:auth (JWT atau Sanctum). Selanjutnya, jalankan php artisan forge:seed-admin untuk membuat akun admin default guna mempermudah login.

Langkah 3

Pembuatan Modul CRUD Relasional

Gunakan flag relasi saat membuat modul CRUD: php artisan forge:add Product --belongsTo=Category --with-factory --upload=image --enum=status:active,inactive --soft-deletes. Ini otomatis menyusun arsitektur relasi induk-anak, validasi upload, enum, dan soft deletes.

Langkah 4

Implementasi Standardisasi API & Pengujian

Eject standardisasi Response menggunakan php artisan forge:response. Daftarkan Policy model via forge:policy. Terakhir, jalankan php artisan forge:test {Model} dan jalankan test-suite project kamu dengan php artisan test.

Langkah 5

Rollback Jika Terjadi Kesalahan

Salah mengetik kolom relasi atau nama model? Tenang! Cukup ketikkan php artisan forge:undo untuk mengembalikan project ke kondisi pristine semula secara otomatis.

One-Shot Install (forge:install)

Fungsi

Menjalankan environment check, auth setup, admin seeder, dan migrate dalam satu perintah.

Kapan Digunakan

Langkah setelah composer require selesai.

Perintah forge:install menjalankan 4 langkah penting secara berurutan dalam satu perintah, sehingga kamu tidak perlu menjalankan satu per satu secara manual.

CLI Command
php artisan forge:install
CLI Command (Dry-Run)
php artisan forge:install --dry-run

Langkah yang Dijalankan

1. doctor — Menjalankan forge:doctor untuk memeriksa environment PHP, database, dan konfigurasi .env.

2. auth — Menjalankan forge:auth untuk setup autentikasi (JWT atau Sanctum) beserta Scramble docs.

3. seed-admin — Menjalankan forge:seed-admin untuk membuat akun admin default.

4. migrate — Menjalankan php artisan migrate untuk menerapkan semua migration ke database.

Gunakan flag --force untuk melewati semua konfirmasi prompt dan langsung menjalankan seluruh langkah secara otomatis.

Artisan CRUD Generator

Fungsi

Men-generate seluruh file CRUD utama (Model, Migration, Controller, Service, Request, Resource) dalam sekali jalan sesuai parameter input.

Kapan Digunakan

Ketika soal ujian meminta kamu membuat modul/entitas database baru (contoh: modul buku, transaksi, produk, dsb.).

Buat file scaffolding lengkap (Model, Migration, Controller, Service, Request, Resource) hanya dengan satu command:

Scaffolding Command
php artisan forge:add Product

Flag Options yang Sangat Penting

--belongsTo=Category

Otomatis menyuntikkan relasi belongsTo di model saat ini, hasMany di model Parent, serta kolom Foreign Key di migration.

--hasMany=Review

Otomatis menyuntikkan relasi hasMany di model saat ini, belongsTo di model Review, serta kolom foreign key di migration tabel Review.

--hasOne=Profile

Menyuntikkan relasi hasOne di model utama dan belongsTo di model Profile, cocok untuk relasi satu-ke-satu.

--belongsToMany=Role

Menyuntikkan relasi belongsToMany di kedua model (many-to-many) beserta pembuatan tabel pivot secara otomatis di migration.

--with-factory

Membuat file Model Factory & Database Seeder sekaligus, terintegrasi otomatis.

--upload=image

Menyisipkan handler upload file (image/pdf/etc.), validasi ukuran file, logic Storage, dan property resource URL.

--web

Mengaktifkan mode Web MVC: Menghasilkan 4 file Blade views (index, create, edit, show) siap pakai yang di-style dengan Tailwind CSS, Controller redirect, dan auto-register ke routes/web.php.

--enum=status:active,inactive

Membuat file Enum baru (app/Enums/Status.php), menambahkan casting Enum ke Model, dan menyiapkan tipe data ENUM di database migration.

--soft-deletes

Menyisipkan trait SoftDeletes ke Model dan mendefinisikan kolom softDeletes() pada file migration database.

--dry-run / --force

--dry-run digunakan untuk preview operasi file sebelum dieksekusi. --force digunakan untuk menimpa file yang sudah ada tanpa konfirmasi prompt.

Setup Authentication

Fungsi

Memasang endpoint register, login, profile me, logout, beserta middleware token security (JWT / Sanctum).

Kapan Digunakan

Di awal ujian jika soal meminta sistem login API atau web view yang mewajibkan proteksi data sensitif.

Buat sistem login, register, profile info (me), dan logout secara instan. Fitur ini akan otomatis menghasilkan endpoint autentikasi modern di dalam folder Controller.

Langkah Setup
  1. Jalankan: php artisan forge:auth
  2. Pilih driver: JWT (tymon/jwt-auth) atau Sanctum (laravel/sanctum).
  3. Ketik Yes/Ya jika ingin sekaligus di-instalkan API docs Scramble secara otomatis.

Auto Guard Config: Middleware generator (forge:add) otomatis membaca metode auth dari config('exam-boots.defaults.auth_method'). Sanctum → auth:sanctum, JWT → auth:api.

File yang Dihasilkan

1. AuthController

API: app/Http/Controllers/Api/DocsAuthController.php

Web: app/Http/Controllers/DocsAuthController.php

Berisi method register(), login(), me(), dan logout().

2. Route Registration

API: routes/api.php

Web: routes/web.php

Didaftarkan otomatis dengan grup route dan middleware protection yang sesuai (auth:sanctum atau auth:api).

Scramble API Docs

Fungsi

Men-generate dokumentasi API interaktif berbasis OpenAPI/Swagger secara otomatis tanpa menulis anotasi manual di controller.

Kapan Digunakan

Ketika kamu ingin menguji endpoint API, atau saat penguji/asesor meminta bukti link dokumentasi API sistem kamu.

Scramble mendokumentasikan API Laravel secara dinamis tanpa anotasi PHPDoc yang rumit. Jika diaktifkan saat setup auth, program akan meng-configure Service Provider agar mendeteksi metode autentikasi secara otomatis.

Cara Mengakses

Nyalakan server lokal kamu dengan perintah: php artisan serve Buka browser ke alamat berikut untuk melihat OpenAPI Docs interaktif: http://localhost:8000/docs/api

Kustomisasi Service Provider

File config scrambler terletak di app/Providers/ScrambleServiceProvider.php. Kamu bisa mengubah deskripsi API, judul dokumen, atau membatasi akses dokumentasi hanya di environment local.

Relasi Interaktif

Fungsi

Menyambungkan dua model Eloquent yang sudah ada secara interaktif, menyuntikkan method relasi, foreign key, dan migrasi tabel pivot.

Kapan Digunakan

Ketika kamu memiliki model yang sudah dibuat dan perlu menghubungkannya lewat relasi 1:1, 1:N, atau N:M di tengah jalannya ujian.

Gunakan `forge:relation` untuk menghubungkan dua model yang sudah ada dengan panduan interaktif terminal. Mendukung relasi One to One, One to Many, dan Many to Many.

CLI Command
php artisan forge:relation

Langkah Menggunakan

1. Tentukan Model Utama (Parent) & Anak (Child):

CLI akan memindai folder app/Models dan meminta kamu memilih model induk (misal: User) dan model anak (misal: Post).

2. Pilih Tipe Relasi:

  • One to Many (1:N): Menambahkan hasMany ke Parent, belongsTo ke Child, dan FK column ke migration.
  • One to One (1:1): Menambahkan hasOne ke Parent, belongsTo ke Child.
  • Many to Many (N:M): Menambahkan belongsToMany ke kedua model, sekaligus men-generate file migration table pivot (contoh: post_user).

ApiResponse Trait

Fungsi

Menyediakan standarisasi format respon JSON agar response sukses dan error (validasi, server error) seragam.

Kapan Digunakan

Saat kamu mulai mengode controller API agar output response JSON rapi dan sesuai standar penilaian penguji.

Standardisasi format JSON API dengan ejecting trait `ApiResponse.php` ke folder `app/Traits/`.

CLI Command
php artisan forge:response

Contoh Kode Kontroler

app/Traits/ApiResponse.php

// Di Controller kamu:

use App\Traits\ApiResponse;


class ProductController extends Controller

{

use ApiResponse;


public function index()

{

return $this->successResponse($products, "Data berhasil diload");

}

}

Policy/Authorization

Fungsi

Membatasi hak akses CRUD model (seperti view, update, delete) berdasarkan ID kepemilikan data user login.

Kapan Digunakan

Ketika soal ujian meminta agar data transaksi/produk hanya boleh diedit/dihapus oleh user yang membuatnya.

Gunakan `forge:policy {Model}` untuk membuat authorization Policy dan auto-register ke dalam array `$policies` di `AuthServiceProvider.php`.

CLI Command
php artisan forge:policy Product

Bagaimana Cara Kerjanya?

1. Pembuatan File: Package men-generate class policy di folder app/Policies/ProductPolicy.php lengkap dengan method standard (viewAny, view, create, update, delete, dsb.).

2. Auto-Registrasi: Jika file app/Providers/AuthServiceProvider.php ada di project kamu, package akan mendeteksi baris protected $policies = [ dan mendaftarkan pemetaan policy secara otomatis.

3. Penerapan Authorization: Kamu bisa menerapkan policy ini di controller kamu:

$this->authorize('update', $product);

Feature Test Skeleton

Fungsi

Membuat automated integration tests (PHPUnit/Pest) untuk menguji seluruh rute API CRUD beserta pengujian unauthorized access.

Kapan Digunakan

Setelah kamu menyelesaikan coding CRUD, untuk menjamin program tidak error dan semua rest API aman saat dinilai penguji.

Buat pengujian otomatis untuk modul CRUD dan endpoint protection. CLI mendukung framework testing bawaan PHPUnit atau Pest PHP.

Generate Command
php artisan forge:test Product # PHPUnit
php artisan forge:test Product --pest # Pest PHP

Cakupan Uji CRUD & Auth

Read: Menguji akses user terautentikasi terhadap endpoint list index.

Create: Menguji pengiriman payload data untuk menyimpan record baru.

Detail: Menguji request pencarian detail data berdasarkan ID.

Update: Menguji pengeditan data yang sudah tersimpan di DB.

Delete: Menguji penghapusan data dan asersi data hilang dari database.

Auth Protection: Memastikan user tanpa token diblokir secara aman dengan status HTTP `401 Unauthorized`.

Rollback & Revert (forge:undo)

Fungsi

Mengembalikan (rollback) seluruh file yang digenerate oleh command terakhir ke kondisi semula menggunakan backup history lokal.

Kapan Digunakan

Saat terjadi kesalahan input flag, salah nama tabel/model, atau terjadi error fatal akibat injection code yang ingin dibatalkan.

Salah ketik nama model atau relasi? Gunakan `forge:undo` untuk mengembalikan file yang dimodifikasi ke versi backup aslinya dan menghapus file yang baru digenerate secara aman.

CLI Command
php artisan forge:undo

Keamanan Tambahan

Tiap kali perintah generator berjalan (seperti forge:add, forge:auth, dll.), package akan menyalin versi asli file target ke folder backup storage/exam-boots/backups/ dan menulis riwayat di history.json. Begitu perintah undo dijalankan, semua backup akan di-restore ke kondisi awal secara presisi.

Prune Backup Tua

Backup yang menumpuk bisa dihapus otomatis dengan flag --prune. Perintah ini akan membersihkan folder backup yang sudah melewati batas retensi (default 3 hari) sesuai konfigurasi exam-boots.backup.retention_days.

CLI Command
php artisan forge:undo --prune
